Introduction & Setup

Understand what Databricks is, where it fits, and how to get your environment running.

What is Databricks?

Databricks is a unified data analytics platform built on top of Apache Spark. It was founded by the original creators of Apache Spark and provides a cloud-based environment for big data processing, machine learning, and analytics at scale.

Databricks implements the Lakehouse architecture — combining the best of data warehouses (structured queries, ACID transactions) with data lakes (low-cost storage, flexibility, ML support).

⚡

Unified Analytics

One platform for data engineering, SQL analytics, data science, and machine learning workflows.

☁️

Cloud-Native

Runs on AWS, Azure, and GCP. Managed clusters spin up automatically — no infrastructure headaches.

🔥

Apache Spark Core

Built on Spark — the industry-standard distributed computing engine for processing petabytes of data.

🏔️

Delta Lake

Open-source storage layer that brings ACID transactions, schema enforcement, and time travel to data lakes.

Architecture Deep Dive

Understand the Control Plane, Data Plane, and how Databricks components work together.

Two-Plane Architecture

Databricks uses a two-plane architecture separating the control infrastructure from your data:

🎛️ Control PlaneDatabricks Cloud (managed by Databricks)
↕ Secure API calls
☁️ Data PlaneYour Cloud Account (AWS/Azure/GCP)
↕
📦 StorageS3 / ADLS / GCS
⚙️ ComputeEC2 / VMs / GCE
🎛️

Control Plane

Hosts the web application, cluster manager, job scheduler, and REST APIs. Managed entirely by Databricks. Stores metadata, configs, and notebook source code.

☁️

Data Plane

Runs in YOUR cloud account. All actual data processing happens here. Your data never leaves your cloud account — Databricks only orchestrates, not stores.

🔵 Key Components Explained
  • Driver: Coordinates all Spark jobs. Runs your main program. One per cluster.
  • Executor: JVM process on each worker node. Runs tasks and stores data in memory/disk.
  • Task: The smallest unit of work. A single executor runs one task at a time per CPU core.
  • Stage: Group of tasks that can run in parallel. Stages are separated by shuffle operations.
  • Job: Complete computation triggered by an action (like .count() or .collect()).
  • DAG: Directed Acyclic Graph — Spark's execution plan. Built lazily and optimized by Catalyst.

💡 Cluster Sizing Tips
  • Start with 4-8 workers, then scale based on job duration
  • Enable autoscaling for unpredictable workloads
  • Use i3 instances (SSD) for shuffle-heavy jobs
  • Use r4/r5 instances (high memory) for ML/large DataFrames
  • Always set auto-termination to avoid runaway costs
  • Use spot/preemptible instances for worker nodes (60-80% savings)

Apache Spark Fundamentals

Understand RDDs, transformations, actions, and the lazy evaluation model.

Spark Execution Model
🔵 Lazy Evaluation — The Core Concept

Spark does NOT execute code immediately. Transformations build a logical plan (DAG). Actions trigger actual execution. This allows Spark to optimize the full computation before running anything.

PySpark — Transformations vs Actions
# ── TRANSFORMATIONS (lazy — nothing runs yet) ──
df = spark.read.parquet("/data/sales")           # lazy
df_filtered = df.filter(df.amount > 100)           # lazy
df_grouped = df_filtered.groupBy("region")        # lazy
df_agg = df_grouped.agg({"amount": "sum"})        # lazy

# ── ACTIONS (trigger execution) ──
count = df_agg.count()              # NOW it runs!
rows = df_agg.collect()            # Bring to driver
df_agg.show(20)                    # Print to console
df_agg.write.parquet("/output")    # Write to storage

# Common transformations
df.select("col1", "col2")          # Select columns
df.filter(df.age > 18)            # Filter rows
df.withColumn("tax", df.price * 0.2) # Add column
df.drop("unwanted_col")            # Drop column
df.distinct()                       # Remove duplicates
df.orderBy("date", ascending=False) # Sort
df.limit(1000)                      # Limit rows
df.cache()                          # Cache in memory

PySpark — DataFrame Operations
from pyspark.sql import functions as F

df = spark.read.delta("/mnt/data/orders")

# Column operations
df.select(
    "order_id",
    "customer_id",
    F.col("amount") * 1.1.alias("amount_with_tax"),
    F.upper(F.col("region")).alias("region_upper"),
    F.to_date("order_date", "yyyy-MM-dd").alias("date")
)

# Aggregations
df.groupBy("region", "product_category").agg(
    F.sum("amount").alias("total_revenue"),
    F.count("order_id").alias("order_count"),
    F.avg("amount").alias("avg_order_value"),
    F.max("order_date").alias("last_order_date"),
    F.countDistinct("customer_id").alias("unique_customers")
).orderBy(F.desc("total_revenue"))

# Conditional expressions
df.withColumn("tier",
    F.when(F.col("amount") > 1000, "Gold")
     .when(F.col("amount") > 500,  "Silver")
     .otherwise("Bronze")
)

# Handling nulls
df.dropna(subset=["customer_id", "amount"])
df.fillna({"region": "Unknown", "amount": 0})
df.filter(F.col("name").isNotNull())

# String functions
df.withColumn("email_domain",
    F.split(F.col("email"), "@").getItem(1))
df.filter(F.col("name").like("%John%"))
df.withColumn("clean_name", F.regexp_replace("name", "[^a-zA-Z ]", ""))
Spark SQL
-- Register DataFrame as temp view
df.createOrReplaceTempView("orders")

-- Run SQL queries
SELECT
    region,
    product_category,
    SUM(amount) AS total_revenue,
    COUNT(*) AS order_count,
    ROUND(AVG(amount), 2) AS avg_order,
    PERCENTILE_APPROX(amount, 0.5) AS median_amount
FROM orders
WHERE order_date >= '2024-01-01'
    AND status NOT IN ('CANCELLED', 'RETURNED')
GROUP BY 1, 2
HAVING total_revenue > 10000
ORDER BY total_revenue DESC
LIMIT 50;

-- CTEs (Common Table Expressions)
WITH monthly_summary AS (
    SELECT
        DATE_TRUNC('month', order_date) AS month,
        customer_id,
        SUM(amount) AS monthly_spend
    FROM orders
    GROUP BY 1, 2
),
top_customers AS (
    SELECT customer_id, SUM(monthly_spend) AS total
    FROM monthly_summary
    GROUP BY 1
    HAVING total > 5000
)
SELECT c.name, t.total
FROM top_customers t
JOIN customers c ON t.customer_id = c.id;

-- Use SQL result back in Python
result_df = spark.sql("""

PySpark — Window Functions
from pyspark.sql import Window
from pyspark.sql import functions as F

# Define window specifications
window_by_customer = Window.partitionBy("customer_id").orderBy("order_date")
window_rolling     = Window.partitionBy("customer_id").orderBy("order_date").rowsBetween(-6, 0)
window_all         = Window.partitionBy("region")

df.withColumn("row_num",    F.row_number().over(window_by_customer)) \
  .withColumn("rank",       F.rank().over(window_by_customer)) \
  .withColumn("dense_rank", F.dense_rank().over(window_by_customer)) \
  .withColumn("running_total", F.sum("amount").over(window_by_customer)) \
  .withColumn("rolling_7day",  F.sum("amount").over(window_rolling)) \
  .withColumn("pct_of_region",
      F.col("amount") / F.sum("amount").over(window_all)) \
  .withColumn("prev_amount",   F.lag("amount", 1).over(window_by_customer)) \
  .withColumn("next_amount",   F.lead("amount", 1).over(window_by_customer)) \
  .withColumn("pct_change",
      (F.col("amount") - F.lag("amount", 1).over(window_by_customer))
      / F.lag("amount", 1).over(window_by_customer))

# Get latest record per group (common pattern)
latest = df.withColumn(
    "rn", F.row_number().over(
        Window.partitionBy("customer_id").orderBy(F.desc("updated_at"))
    )
).filter(F.col("rn") == 1).drop("rn")
PySpark — User Defined Functions
from pyspark.sql.functions import udf, pandas_udf
from pyspark.sql.types import StringType, DoubleType
import pandas as pd

# Standard UDF (slower — Python serialization per row)
def classify_amount(amount):
    if amount is None: return "Unknown"
    if amount > 1000: return "High"
    if amount > 100:  return "Medium"
    return "Low"

classify_udf = udf(classify_amount, StringType())
df.withColumn("tier", classify_udf(F.col("amount")))

# Pandas UDF (MUCH faster — vectorized via Apache Arrow)
@pandas_udf(DoubleType())
def apply_discount(amounts: pd.Series) -> pd.Series:
    return amounts * 0.9  # Vectorized operation

df.withColumn("discounted", apply_discount(F.col("amount")))

# Register UDF for SQL use
spark.udf.register("classify_amount_sql", classify_amount, StringType())
spark.sql("SELECT classify_amount_sql(amount) FROM orders")

# ⚠️ Best Practices:
# 1. Prefer built-in functions (F.*) over UDFs — they run in JVM
# 2. Use Pandas UDFs over Python UDFs for better performance
# 3. Never use UDFs for simple transformations (null handling, math, strings)
Core›Delta Lake
06

Delta Lake

ACID transactions, time travel, schema evolution, and DML operations on data lakes.

What Makes Delta Lake Special?
🔒

ACID Transactions

Full transactional support. Concurrent reads and writes without corruption. No more partial writes or dirty reads.

⏱️

Time Travel

Query any historical version of your data. Roll back mistakes instantly. Audit every change.

📋

Schema Enforcement

Prevents bad data from corrupting your tables. Schema evolution lets you safely add columns.

🗂️

DML Operations

Full UPDATE, DELETE, and MERGE (upsert) support — impossible with plain Parquet files.

🔵 Delta Log — How It Works

Delta Lake stores data as Parquet files + a transaction log (_delta_log directory). Every operation is recorded as a JSON commit. Spark reads the log to reconstruct the current and historical state of the table. This is how time travel and ACID transactions work.

PySpark / SQL — Delta Operations
# ── CREATE DELTA TABLE ──
df.write.format("delta") \
    .mode("overwrite") \
    .partitionBy("year", "month") \
    .save("/mnt/delta/orders")

# Or create managed table
df.write.saveAsTable("default.orders")

# ── READ DELTA ──
df = spark.read.format("delta").load("/mnt/delta/orders")

# ── TIME TRAVEL ──
# Read specific version
df_v5 = spark.read.format("delta") \
    .option("versionAsOf", 5) \
    .load("/mnt/delta/orders")

# Read at specific timestamp
df_yesterday = spark.read.format("delta") \
    .option("timestampAsOf", "2024-01-15 10:00:00") \
    .load("/mnt/delta/orders")

# ── DML OPERATIONS ──
from delta.tables import DeltaTable

delta_table = DeltaTable.forPath(spark, "/mnt/delta/orders")

# UPDATE
delta_table.update(
    condition="status = 'PENDING'",
    set={"status": "'PROCESSING'"}
)

# DELETE
delta_table.delete("order_date < '2020-01-01'")

# MERGE (upsert)
delta_table.alias("target").merge(
    new_data.alias("source"),
    "target.order_id = source.order_id"
).whenMatchedUpdate(set={
    "status": "source.status",
    "updated_at": "source.updated_at"
}).whenNotMatchedInsertAll().execute()

# ── HISTORY & RESTORE ──
delta_table.history().show(truncate=False)   # View all commits
delta_table.restoreToVersion(3)               # Roll back to version 3

# ── OPTIMIZE & VACUUM ──
delta_table.optimize().executeCompaction()     # Compact small files
delta_table.optimize().executeZOrderBy("customer_id", "order_date") # Co-locate
delta_table.vacuum(168)                       # Delete files > 7 days old

Delta Live Tables (DLT)
Python — Delta Live Tables
import dlt
from pyspark.sql import functions as F

# ── BRONZE: Raw ingestion ──
@dlt.table(comment="Raw orders from S3")
def bronze_orders():
    return spark.readStream.format("cloudFiles") \
        .option("cloudFiles.format", "json") \
        .load("s3://raw-bucket/orders/")

# ── SILVER: Cleaned & validated ──
@dlt.table
@dlt.expect_or_drop("valid_amount", "amount > 0")
@dlt.expect_or_drop("valid_customer", "customer_id IS NOT NULL")
def silver_orders():
    return dlt.read_stream("bronze_orders") \
        .withColumn("order_date", F.to_date("order_timestamp")) \
        .withColumn("amount", F.col("amount").cast("double")) \
        .drop("_rescued_data")

# ── GOLD: Business aggregates ──
@dlt.table(comment="Daily revenue by region")
def gold_daily_revenue():
    return dlt.read("silver_orders") \
        .groupBy("order_date", "region") \
        .agg(
            F.sum("amount").alias("revenue"),
            F.count("*").alias("order_count")
        )
💡 DLT Quality Expectations
  • @dlt.expect("msg", "condition"): Track violations as metrics but keep data
  • @dlt.expect_or_drop("msg", "condition"): Drop rows that fail
  • @dlt.expect_or_fail("msg", "condition"): Fail the entire pipeline on violation
Intermediate›Structured Streaming
08

Structured Streaming

Process real-time data streams with exactly-once semantics and fault tolerance.

🔵 Streaming Mental Model

Think of a stream as an unbounded table that keeps growing. Spark Structured Streaming treats streaming data the same as batch DataFrames — same API, but with checkpointing for fault tolerance and exactly-once processing guarantees.

PySpark — Kafka Streaming
# Read from Kafka
stream_df = spark.readStream \
    .format("kafka") \
    .option("kafka.bootstrap.servers", "broker1:9092,broker2:9092") \
    .option("subscribe", "orders-topic") \
    .option("startingOffsets", "latest") \
    .load()

# Deserialize JSON payload
from pyspark.sql.types import StructType, StringType, DoubleType
from pyspark.sql.functions import from_json, col

schema = StructType() \
    .add("order_id", StringType()) \
    .add("amount", DoubleType()) \
    .add("timestamp", StringType())

parsed = stream_df \
    .select(from_json(col("value").cast("string"), schema).alias("data")) \
    .select("data.*")

# Windowed aggregation with watermark
from pyspark.sql.functions import window, to_timestamp

windowed = parsed \
    .withColumn("event_time", to_timestamp("timestamp")) \
    .withWatermark("event_time", "10 minutes") \  # Handle late data
    .groupBy(window("event_time", "5 minutes")) \
    .agg(F.sum("amount").alias("revenue_5min"))

# Write stream to Delta table
query = windowed.writeStream \
    .format("delta") \
    .outputMode("append") \
    .option("checkpointLocation", "/mnt/checkpoints/orders") \
    .trigger(processingTime="30 seconds") \
    .toTable("streaming.order_windows")

query.awaitTermination()

# Trigger options
# .trigger(processingTime="30 seconds") — micro-batch every 30s
# .trigger(once=True)           — run once then stop
# .trigger(availableNow=True)   — process all available, then stop
# .trigger(continuous="1 second") — true continuous (low latency)

import mlflow

def objective(params):
    with mlflow.start_run(nested=True):

Advanced Design Patterns
🟣 Production Checklist
  • Idempotency: Every pipeline run should produce the same result. Use MERGE over INSERT.
  • Partitioning Strategy: Partition by low-cardinality columns used in filters (date, region).
  • Z-Ordering: Apply to high-cardinality columns queried frequently (user_id, order_id).
  • Vacuum Schedule: Run VACUUM weekly. Default 7-day retention for time travel.
  • OPTIMIZE Schedule: Run after large writes. Target 128MB+ file sizes.
  • Job Clusters: Use job clusters for production — cheaper, fresh environment, auto-terminate.
  • Separate Environments: dev → staging → prod catalogs in Unity Catalog.
  • CI/CD: Use Databricks Asset Bundles (DABs) with GitHub Actions or Azure DevOps.
  • Monitoring: Track DLT expectations, MLflow metrics, workflow SLAs.
  • Cost Management: Spot instances for workers, right-size clusters, instance pools.
